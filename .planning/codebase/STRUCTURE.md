## Directory Purposes

**frontend/:**
- Purpose: React application source code and assets
- Contains: React components, views, styles, configuration
- Key files: `Dockerfile` (node:24-alpine base, runs `npm run dev` on port 5173)
- Expected structure: `src/`, `public/`, `vite.config.js`, `tsconfig.json`, `package.json`

**backend/:**
- Purpose: Hono API server and business logic
- Contains: Route handlers, controllers, middleware, database operations
- Key files: `Dockerfile` (currently empty - needs completion)
- Expected structure: `src/`, `prisma/`, `tsconfig.json`, `package.json`
- Local mounts: `/app/src/uploads` for persistent file storage across container restarts

**.planning/codebase/:**
- Purpose: Generated architecture and development documentation
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, STACK.md, INTEGRATIONS.md
- Auto-generated by GSD analysis tools

**Root configuration:**
- `compose.dev.yml`: Docker Compose service definitions (frontend, backend)
- `Makefile`: Development commands (`make dev`, `make dev-build`, `make dev-up`)
- `.env`: Environment variables file (created locally from `.env.example`, not committed)

## Key File Locations

**Entry Points:**
- `frontend/Dockerfile`: Frontend container start command → `npm run dev` (runs on port 5173)
- `backend/Dockerfile`: Backend container start command (empty - needs implementation)
- `compose.dev.yml`: Orchestrates both services with volume mounts and networking
- `Makefile`: `dev` target initiates full development environment

**Configuration:**
- `.env`: Runtime environment variables (database URL, API keys, etc.)
- `.env.example`: Template showing required environment variables
- `compose.dev.yml`: Docker service definitions and volume/port mappings

**Core Logic (Expected Locations):**
- `frontend/src/`: React components, pages, utilities, hooks
- `backend/src/`: Hono routes, controllers, middleware, services
- `backend/prisma/schema.prisma`: Database schema definition (standard Prisma location)

**Testing (Expected Locations):**
- `frontend/src/**/*.test.ts` or `frontend/src/**/*.spec.ts`: Frontend unit tests
- `backend/src/**/*.test.ts` or `backend/src/**/*.spec.ts`: Backend unit tests
- `frontend/__tests__/`: Component and integration tests (alternative location)
- `backend/__tests__/`: API and integration tests (alternative location)

**Database:**
- `backend/prisma/schema.prisma`: Prisma schema (expected)
- `backend/prisma/migrations/`: Database migration history (expected)
- Database connection: Configured via environment variable, accessible through Prisma Studio on port 5555

## Naming Conventions

**Files:**
- `*.tsx`: React components (TypeScript + JSX)
- `*.ts`: TypeScript utilities, services, middleware
- `*.test.ts` or `*.spec.ts`: Test files
- `Dockerfile`: Container configuration

**Directories:**
- `src/`: Source code root
- `src/components/`: React components (frontend)
- `src/routes/`: Route handlers (backend, expected)
- `src/services/`: Business logic and utilities
- `src/uploads/`: User-generated file storage (backend)
- `prisma/`: Prisma ORM configuration and migrations
- `__tests__/`: Test suite directory (alternative to co-located tests)

## Where to Add New Code

**New Feature:**
- Primary code: `frontend/src/` (React components) and `backend/src/` (API routes/handlers)
- Tests: Co-located `*.test.ts` or in `__tests__/` directory
- Database schema: Update `backend/prisma/schema.prisma`, create migration via `npx prisma migrate dev --name feature_name`

**New Component/Module:**
- Frontend component: `frontend/src/components/[ComponentName]/`
- Backend service: `backend/src/services/[ServiceName].ts`
- Backend route: `backend/src/routes/[routeName].ts`

**Utilities and Helpers:**
- Frontend: `frontend/src/utils/` or `frontend/src/lib/`
- Backend: `backend/src/utils/` or `backend/src/lib/`
- Shared (if monorepo utilities): `packages/` (not yet present)

**Styling:**
- Frontend: Co-located CSS/SCSS files or in `frontend/src/styles/`
- Component-scoped styles: `frontend/src/components/[Component]/[Component].module.css`

## Special Directories

**node_modules/:**
- Purpose: Installed npm dependencies
- Generated: Yes (via `npm install`)
- Committed: No (excluded via `.gitignore`)
- Volume mount: Separate volume in Docker to preserve between development sessions

**.git/:**
- Purpose: Git version control repository
- Committed: Yes (core repository metadata)
- Not to be modified directly

**src/uploads/ (backend):**
- Purpose: Persistent file storage for user uploads
- Generated: Yes (at runtime by backend)
- Committed: No
- Volume mount: Explicitly mounted in `compose.dev.yml` to survive container restarts

**.env:**
- Purpose: Runtime configuration and secrets
- Generated: Manually from `.env.example`
- Committed: No (never commit secrets)
- Structure: KEY=VALUE pairs, one per line
